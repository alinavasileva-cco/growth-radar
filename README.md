Радар компаний, которым потенциально нужен коммерческий консалтинг, бизнес-трекинг или помощь со структурой роста.

## Цель

Сформировать рабочую базу до 1000 российских компаний для последовательного выхода в течение месяца. Компании группируются по рынкам на основании основного ОКВЭД: FMCG, Auto, IT & Telecom, E-commerce & Retail, Healthcare, Construction & Real Estate и другим сегментам.

## Как устроен отбор

Система:

1. собирает новости и публичные сигналы роста, падения и управленческого найма;
2. сопоставляет бренд с проверенной карточкой юридического лица;
3. проверяет ИНН, ОГРН, собственника или директора, ОКВЭД и финансовые показатели;
4. относит компанию к практическому сегменту рынка;
5. отбирает компании по одному из критериев:
   - выручка 30–150 млн ₽;
   - снижение выручки от 15%;
   - новый активный бизнес до двух лет;
6. отдельно проверяет наличие прямого контакта ЛПР.

Общий адрес отдела продаж или маркетинга не считается прямым контактом ЛПР.

## Статусы

- `outreach_ready` — компания прошла отбор и найден прямой контакт ЛПР;
- `needs_direct_contact` — компания и финансы проверены, но прямой контакт пока не найден;
- `needs_identity_check` — требуется подтвердить собственника или директора;
- `needs_enrichment` — найден сигнал, но карточка компании ещё не собрана;
- `excluded` — компания не соответствует критериям.

## Выходные файлы

- `data/company_profiles.csv` — проверенные карточки компаний;
- `data/company_pipeline.csv` — полный рабочий список;
- `data/outreach_ready.csv` — только компании, которым уже можно писать;
- `mvp-results/review_1000_companies.csv` — база для ручной оценки;
- `mvp-results/segment_summary.csv` — количество компаний по сегментам и статусам.

## Автоматизация

GitHub Actions запускает сбор ежедневно в 09:30 по московскому времени и обновляет файлы в `mvp-results`.

## Быстрый запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m playwright install chromium
python main.py
```

На первом запуске будет создана база `data/growth_radar.db`.
